# Writing a changelog entry

When asked to write a changelog:

1. Group changes under **Added**, **Changed**, **Fixed**, **Removed**.
2. One bullet per user-visible change, in the imperative ("Add …", "Fix …").
3. Omit internal refactors unless they change behavior.
4. Link issues/PRs where available.
